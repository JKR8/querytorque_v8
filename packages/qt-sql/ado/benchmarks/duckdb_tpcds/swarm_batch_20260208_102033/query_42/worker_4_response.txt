```sql
SELECT
  dt.d_year,
  item.i_category_id,
  item.i_category,
  SUM(ss_ext_sales_price)
FROM store_sales
JOIN (
  SELECT d_date_sk, d_year
  FROM date_dim
  WHERE d_moy = 11 AND d_year = 2002
) AS dt ON store_sales.ss_sold_date_sk = dt.d_date_sk
JOIN (
  SELECT i_item_sk, i_category_id, i_category
  FROM item
  WHERE i_manager_id = 1
) AS item ON store_sales.ss_item_sk = item.i_item_sk
GROUP BY
  dt.d_year,
  item.i_category_id,
  item.i_category
ORDER BY
  SUM(ss_ext_sales_price) DESC,
  dt.d_year,
  item.i_category_id,
  item.i_category
LIMIT 100
```

**Changes**: Converted implicit joins to explicit JOINs and pre-filtered dimension tables using subqueries (instead of CTEs) to allow DuckDB maximum freedom to push filters and reorder joins. This isolates the filtered dimensions early while avoiding materialization overhead.

**Expected speedup**: 1.2-1.5x from reduced join cardinality and better optimizer freedom.
