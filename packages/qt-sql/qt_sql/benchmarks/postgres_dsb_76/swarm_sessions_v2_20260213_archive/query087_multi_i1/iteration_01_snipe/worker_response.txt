## Part 1: Modified Logic Tree

```
[=] original_query (semantics preserved)
  [+] filtered_date (CTE) - pre-filter date_dim
  [+] filtered_customer (CTE) - pre-filter customer
  [+] store_sales_join (CTE) - distinct store combinations
  [+] catalog_sales_join (CTE) - distinct catalog combinations  
  [+] web_sales_join (CTE) - distinct web combinations
  [~] set_difference (CTE) - changed from NOT EXISTS to anti-join with UNION
  [=] main_query - COUNT(*)
```

**Changes:** The rewrite consolidates the anti-join logic by checking against a UNION of catalog and web sales in a single NOT EXISTS, reducing the number of anti-join probes from 2 to 1. It preserves explicit JOIN syntax and pre-filtered dimension CTEs.

**Expected speedup:** ~2.2x. The key bottleneck was repeated anti-join probes (2 per row). By unifying the exclusion set into a single UNION, we cut the anti-join work in half while maintaining the same semantics. The configuration changes favor hash joins over nested loops for the larger fact table joins.
