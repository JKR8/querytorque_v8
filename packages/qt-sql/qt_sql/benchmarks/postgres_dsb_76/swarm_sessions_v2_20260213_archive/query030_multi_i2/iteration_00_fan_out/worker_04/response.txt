### Part 1: Modified Logic Tree

```
QUERY: (single statement)
├── [CTE] filtered_dates  [=]
├── [CTE] filtered_items  [=]
├── [CTE] filtered_addresses_current  [=]
├── [CTE] filtered_addresses_returning  [~] (removed state filter to preserve original semantics)
│   ├── SCAN (customer_address)
│   └── OUTPUT (ca_address_sk, ca_state)
├── [CTE] web_returns_joined  [~] (uses filtered_addresses_returning without state filter)
│   ├── SCAN (web_returns, filtered_dates (join), filtered_addresses_returning (join), filtered_items (join))
│   ├── JOIN (wr_returned_date_sk = d_date_sk)
│   ├── JOIN (wr_returning_addr_sk = ca_address_sk)
│   ├── JOIN (wr_item_sk = i_item_sk)
│   ├── FILTER (wr_return_amt / wr_return_quantity BETWEEN 242 AND 271)
│   └── OUTPUT (wr_returning_customer_sk, ca_state, wr_reason_sk, wr_return_amt)
├── [CTE] cte_aggregated  [~] (renamed from customer_total_return; same logic)
│   ├── SCAN (web_returns_joined)
│   ├── AGG (GROUP BY wr_returning_customer_sk, ca_state, wr_reason_sk)
│   └── OUTPUT (ctr_customer_sk, ctr_state, ctr_reason_sk, ctr_total_return)
├── [CTE] state_avg  [+] (decorrelates the subquery)
│   ├── SCAN (cte_aggregated)
│   ├── AGG (GROUP BY ctr_state)
│   └── OUTPUT (ctr_state, state_avg_threshold)
└── [MAIN] main_join  [~] (joins CTEs instead of correlated subquery)
    ├── SCAN (cte_aggregated AS ctr1 (join), state_avg (join), customer (join), filtered_addresses_current (join))
    ├── JOIN (ctr1.ctr_state = state_avg.ctr_state)
    ├── JOIN (ctr1.ctr_customer_sk = c_customer_sk)
    ├── JOIN (ca_address_sk = c_current_addr_sk)
    ├── FILTER (ctr1.ctr_total_return > state_avg.state_avg_threshold)
    ├── FILTER (ctr1.ctr_reason_sk IN (20, 36))
    ├── FILTER (c_birth_year BETWEEN 1985 AND 1991)
    ├── SORT (same 13‑column ORDER BY)
    └── OUTPUT (same 13 columns)
```

Changes: The rewrite follows the target logical tree but keeps `filtered_addresses_returning` without a state filter to preserve exact semantic equivalence (returning address state is not restricted, only current address state). The correlated subquery is decorrelated into a separate `state_avg` CTE that computes the per‑state threshold once, eliminating the per‑row execution observed in the original plan.

Expected speedup: ~2–3× due to elimination of correlated subquery loops and better dimension‑table cardinality estimates from prefetching.
