### Part 1: Modified Logic Tree

```
QUERY: (single statement)
├── [CTE] customer_total_return -> returns_cte [~] (renamed, made MATERIALIZED, kept same scan)
│   ├── SCAN (catalog_returns, date_dim (comma join), customer_address (comma join)) [=]
│   ├── JOIN (cr_returned_date_sk = d_date_sk) [=]
│   ├── JOIN (cr_returning_addr_sk = ca_address_sk) [=]
│   ├── FILTER (d_year = 2000) [=]
│   ├── AGG (GROUP BY cr_returning_customer_sk, ca_state) [=]
│   └── OUTPUT (ctr_customer_sk, ctr_state, ctr_total_return) [=]
├── [CTE] state_avg_cte [+] (new CTE for per-state averages)
│   ├── SCAN (returns_cte)
│   ├── AGG (GROUP BY ctr_state)
│   └── OUTPUT (ctr_state, state_avg)
└── [MAIN] main_query -> main_decomposed [~] (correlated subquery replaced with JOIN to state_avg_cte)
    ├── SCAN (returns_cte AS ctr1 (join), state_avg_cte (join), customer (join), customer_address (join)) [~]
    ├── JOIN (ctr1.ctr_state = state_avg_cte.ctr_state AND ctr1.ctr_total_return > state_avg_cte.state_avg * 1.2)
    ├── JOIN (ctr1.ctr_customer_sk = c_customer_sk)
    ├── JOIN (ca_address_sk = c_current_addr_sk AND ca_state = 'IA')
    ├── SORT (same 16-column ORDER BY) [=]
    ├── LIMIT 100 [=]
    └── OUTPUT (same 15 customer/address columns + ctr_total_return) [=]
```

**Changes**: Materialized the base returns CTE and added a separate CTE for state averages, replacing the correlated subquery with a JOIN. This prevents duplicate computation and decorrelates the per-state threshold comparison.

**Expected speedup**: ~3-4x (eliminates per-row correlated subquery execution and redundant fact table scans).
