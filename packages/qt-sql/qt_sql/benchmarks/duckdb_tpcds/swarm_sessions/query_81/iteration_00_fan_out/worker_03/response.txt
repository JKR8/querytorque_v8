### Part 1: Modified Logic Tree

```
QUERY: (single statement)
├── [CTE] ctr_all [=] Cost: 50% Rows: ~276K — Aggregate 2002 catalog return amounts by returning customer and return-address state.
│   ├── SCAN (catalog_returns, date_dim, customer_address (return address))
│   ├── JOIN (cr_returned_date_sk = d_date_sk)
│   ├── JOIN (cr_returning_addr_sk = ca_address_sk)
│   ├── FILTER (d_year = 2002)
│   ├── AGG (GROUP BY cr_returning_customer_sk, ca_state)
│   └── OUTPUT (ctr_customer_sk, ctr_state, ctr_total_return)
├── [CTE] ctr_state_avg [+] Cost: 1% Rows: 51 — State-level averages from ctr_all.
│   ├── SCAN (ctr_all)
│   ├── AGG (GROUP BY ctr_state)
│   └── OUTPUT (ctr_state, state_avg)
├── [CTE] ctr_ca [+] Cost: 48% Rows: 5,054 — Filter to CA customers and enrich with current address/customer details.
│   ├── SCAN (ctr_all, customer, customer_address (current))
│   ├── JOIN (ctr_all.ctr_customer_sk = c_customer_sk)
│   ├── JOIN (c_current_addr_sk = ca_address_sk)
│   ├── FILTER (ca_state = 'CA')
│   └── OUTPUT (ctr_customer_sk, ctr_state, ctr_total_return, c_customer_id, c_salutation, c_first_name, c_last_name, ca_street_number, ca_street_name, ca_street_type, ca_suite_number, ca_city, ca_county, ca_state, ca_zip, ca_country, ca_gmt_offset, ca_location_type)
└── [MAIN] main_query [~] Cost: 1% Rows: 100 — Join state averages, apply 1.2x filter, order, limit.
    ├── SCAN (ctr_ca, ctr_state_avg)
    ├── JOIN (ctr_ca.ctr_state = ctr_state_avg.ctr_state)
    ├── FILTER (ctr_total_return > state_avg * 1.2)
    ├── SORT (c_customer_id, c_salutation, c_first_name, c_last_name, ca_street_number, ca_street_name, ca_street_type, ca_suite_number, ca_city, ca_county, ca_state, ca_zip, ca_country, ca_gmt_offset, ca_location_type, ctr_total_return)
    └── OUTPUT (16 columns per contract)
```

**Changes:** The rewrite splits the original correlated subquery into two CTEs—one for per-state averages (`ctr_state_avg`) and one for California customers with current address details (`ctr_ca`). The main query then joins these explicitly, replacing the correlated filter with a hash join.

**Expected speedup:** 1.2–1.4x (decorrelation eliminates per-row subquery execution; CTE specialization reduces redundant scanning of the base aggregate).
