### Part 1: Modified Logic Tree

```
QUERY: (single statement)
├── [CTE] ctr_with_avg  [~]  Cost: 60%  Rows: ~276K  — Aggregate 2002 catalog return amounts with tax by returning customer and return-address state, compute state average via window (eliminating correlated subquery).
│   ├── SCAN (catalog_returns, date_dim (join), customer_address (join))
│   ├── JOIN (cr_returned_date_sk = d_date_sk)
│   ├── JOIN (cr_returning_addr_sk = ca_address_sk)
│   ├── FILTER (d_year = 2002)
│   ├── AGG (GROUP BY cr_returning_customer_sk, ca_state)  [=]
│   ├── WINDOW (PARTITION BY ca_state)  [+] — Compute state_avg over aggregated ctr_total_return
│   └── OUTPUT (ctr_customer_sk, ctr_state, ctr_total_return, state_avg)
└── [MAIN] main_query  [~]  Cost: 40%  Rows: ~100  — Join aggregated returns with customer and current address, filter CA and >120% state average, order and limit.
    ├── SCAN (ctr_with_avg AS ctr1 (join), customer_address (join), customer (join))
    ├── JOIN (ca_address_sk = c_current_addr_sk)
    ├── JOIN (ctr1.ctr_customer_sk = c_customer_sk)
    ├── FILTER (ctr_total_return > state_avg * 1.2)  [~] — Now uses pre‑computed state_avg from CTE (no subquery)
    ├── FILTER (ca_state = 'CA')  [=]
    ├── SORT (c_customer_id ASC, c_salutation ASC, c_first_name ASC, c_last_name ASC, ca_street_number ASC, ca_street_name ASC, ca_street_type ASC, ca_suite_number ASC, ca_city ASC, ca_county ASC, ca_state ASC, ca_zip ASC, ca_country ASC, ca_gmt_offset ASC, ca_location_type ASC, ctr_total_return ASC)  [=]
    └── OUTPUT (c_customer_id, c_salutation, c_first_name, c_last_name, ca_street_number, ca_street_name, ca_street_type, ca_suite_number, ca_city, ca_county, ca_state, ca_zip, ca_country, ca_gmt_offset, ca_location_type, ctr_total_return)
```

**Changes**: Replaced the correlated subquery with a window function that computes the state average in the same CTE scan, eliminating N² subquery overhead while preserving exact semantics.

**Expected speedup**: ~1.2–1.5× from eliminating correlated subquery execution and enabling better predicate pushdown (though CA filter cannot be pushed into CTE).
