### Part 1: Modified Logic Tree

```
QUERY:
└── [~] MAIN
    ├── [+] CTE date_cte
    ├── [+] CTE time_cte
    ├── [+] CTE ship_cte
    ├── [+] CTE web_branch
    ├── [+] CTE catalog_branch
    ├── [+] CTE union_cte (implicit via UNION ALL in main)
    ├── [=] AGG (GROUP BY w_warehouse_name, w_warehouse_sq_ft, w_city, w_county, w_state, w_country, ship_carriers, year)
    ├── [=] SORT (w_warehouse_name ASC)
    ├── [=] LIMIT 100
    └── [=] OUTPUT (33 columns exactly as in original)
```

**Changes**: Isolated dimension table filters (date/time/ship_mode) into reusable CTEs, then created separate channel branches that join these filtered dimension CTEs directly with fact tables, maintaining the original UNION ALL structure. This prevents Cartesian explosion by joining dimension CTEs sequentially with fact tables.

**Expected speedup**: 1.5-2x from reduced dimension table scans (each dimension scanned once instead of twice) and predicate pushdown into small hash tables for fact table joins.
