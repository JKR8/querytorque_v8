Looking at the query, I see a correlated subquery with an aggregate comparison. This matches **QT-OPT-002** from the patterns - we can convert the correlated subquery to a pre-computed CTE. The query also has date filtering which suggests **QT-OPT-003** (Date CTE Isolation).

Key observation: The subquery calculates `1.3 * AVG(ws_ext_discount_amt)` for each item within the same date range. Currently it's executed for every row in the outer query. We can precompute this in a CTE.
