**Key insight**: The query joins `store_returns` with `customer_demographics`, but `customer_demographics` itself is only joined with `customer` (via `c_current_cdemo_sk`). We can pre-filter `store_returns` by the available customer demographic keys from the filtered customer set, reducing the large fact table scan.

The `store_returns` table is joined on `sr_cdemo_sk = cd.cd_demo_sk`, and `customer_demographics` is joined to `customer` on `cd_demo_sk = c.c_current_cdemo_sk`. This means we can potentially filter `store_returns` by getting the `cd_demo_sk` values from customers that have already been filtered by address, household demographics, and income band.
